# 14. Glossary

| Term | Meaning |
|---|---|
| **Action card / action message** | A structured message to a worker carrying one tap-to-complete button (e.g. re-upload this card) — vs a free-text bubble |
| **Attention block** | The dashboard's triage list of items most needing a human today; dismissed items return after a cooldown |
| **Audit trail** | The permanent, unchangeable record of what happened, when, who, and why — per worker, card, and (admins) recruiter |
| **Card** | Any credential document: CSCS, CPCS, NPORS, IPAF, PASMA and the rest, plus certificates |
| **Client** | A company you supply workers to; owns projects |
| **Compliance status** | Green/amber/red verdict on whether a worker's required documents exist, are verified, and are in date |
| **CSCS** | The Construction Skills Certification Scheme card — the industry's baseline competence card; a valid verified CSCS is the gate to Site Strong |
| **Framework** | An umbrella agreement with a client holding pre-agreed terms; projects created under it inherit those terms |
| **Hold** | Parking a card in the verification queue without deciding — for when you need a colleague or the worker first |
| **New Hand / Site Starter / Site Ready / Site Strong / Site Pro** | The five profile quality tiers, lowest to highest ([1.3](01-concepts.md)) |
| **Placement** | The record connecting a worker to a project, with dates — the source of real site history |
| **Quick Approve** | The queue's grid view for batch-approving clean, high-confidence cards; anything doubtful must be inspected individually |
| **Re-upload request** | Asking a worker to submit a document again, with a plain-words reason; arrives as an action card |
| **Right to Work (RTW)** | A worker's legal permission to work in the UK; non-UK/Irish workers evidence it via a government **share code** |
| **Share code** | The code from the government's online service that lets an employer check a worker's RTW status; codes expire |
| **Site view** | A project's deployment picture: who's placed there and their compliance state |
| **Tier / completeness score** | The 0–100 score (recruiter-visible) summarising a profile's verified completeness, and the five-band tier built on it (worker-visible) |
| **Verification / verified** | A recruiter's recorded judgment that a document is genuine, current, and correctly read — the only state in which a document counts for anything |
| **Verification queue** | The ordered list of uploaded documents awaiting that judgment |
