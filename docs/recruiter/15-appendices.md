# Appendices

## A. Keyboard shortcuts

| Where | Key | Action |
|---|---|---|
| Anywhere | ++slash++ | Focus search |
| Verification queue | ++a++ / ++h++ / ++r++ | Approve / Hold / Reject the open card |
| Verification queue | ++j++ / ++k++ | Next / previous card |
| Verification queue | ++g++ | Grid (Quick Approve) view |

## B. Tier bands — quick reference

| Score | Tier | In one line |
|---|---|---|
| 0–19 | **New Hand** | Nothing verified yet |
| 20–34 | **Site Starter** | First core verification landed |
| 35–69 | **Site Ready** | Identity + CSCS verified — shortlist-ready |
| 70–84 | **Site Strong** | All four pillars verified: ID, CSCS, 2 cards, experience |
| 85–100 | **Site Pro** | Locked — later phase (references) |

Boundary scores belong to the higher tier. No valid CSCS → maximum 65 → Site Strong unreachable. Recruiters see scores; workers see tiers only.

## D. Status colour language

| Colour | Everywhere it appears, it means |
|---|---|
| **Green** | Good / verified / compliant / resolved |
| **Amber** | Attention soon / expiring / awaiting reply / prompt |
| **Red** | Action required now / expired / needs you |
| **Grey** | Inactive / neutral / historical |
